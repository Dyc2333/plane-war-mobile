# 飞机大战射击游戏 - 架构设计文档

## 一、整体架构

采用经典游戏循环模式 + 面向对象设计，模块化分层架构：

```
┌─────────────────────────────────────────────────────────┐
│                      游戏主循环                          │
│                   (Game Loop)                           │
└─────────────────────────────────────────────────────────┘
                            │
        ┌───────────────────┼───────────────────┐
        ▼                   │                   ▼
┌──────────────┐      ┌──────────────┐   ┌──────────────┐
│  输入处理层  │      │   更新层     │   │   渲染层     │
│   Input      │      │   Update     │   │   Render     │
└──────────────┘      └──────────────┘   └──────────────┘
        │                   │                   │
        └───────────────────┼───────────────────┘
                            ▼
                   ┌──────────────┐
                   │  游戏状态机  │
                   │   State      │
                   └──────────────┘
```

## 三、核心类设计

### 3.1 Game（游戏主类）

**职责**：管理游戏全局状态、主循环、场景切换

```javascript
class Game {
  constructor(canvas)
  init()                          // 初始化游戏
  start()                         // 开始游戏
  pause()                         // 暂停游戏
  resume()                        // 继续游戏
  gameOver()                      // 游戏结束
  gameLoop(timestamp)            // 游戏主循环
  update(dt)                     // 更新所有实体
  draw()                         // 渲染所有内容
  spawnEnemy()                   // 生成敌机
  spawnItem(x, y, type)          // 生成道具
  useBomb()                      // 使用炸弹
  addScore(points)              // 增加分数
}
```

**状态**：
- gameState: 'menu' | 'playing' | 'paused' | 'gameover'
- score: number
- highScore: number
- bombCount: number
- enemies: Enemy[]
- bullets: Bullet[]
- items: Item[]
- particles: Particle[]
- player: Player
- difficulty: number

## 四、碰撞检测系统

使用矩形碰撞检测（AABB）：

```javascript
function checkCollision(a, b) {
  return a.x < b.x + b.width &&
         a.x + a.width > b.x &&
         a.y < b.y + b.height &&
         a.y + a.height > b.y;
}
```

**检测分类**：
1. 玩家子弹 vs 敌机
2. 敌机子弹/本体 vs 玩家
3. 玩家 vs 道具

## 五、游戏循环架构

```javascript
function gameLoop(timestamp) {
  // 计算delta time
  const dt = Math.min((timestamp - lastTime) / 1000, 0.1);

  // 扩展处理（处理时间累积）
  accumulator += dt;
  while (accumulator >= FIXED_DT) {
    update(FIXED_DT);
    accumulator -= FIXED_DT;
  }

  // 渲染（使用插值实现平滑）
  draw(accumulator / FIXED_DT);

  lastTime = timestamp;
  requestAnimationFrame(gameLoop);
}
```

## 七、性能优化考虑

1. **对象池**：子弹、粒子使用对象池减少GC
2. **视锥剔除**：移出屏幕的实体自动清理
3. **时间分片**：复杂的AI计算分散到多帧
4. **Canvas优化**：使用willReadFrequently

## 八、扩展性设计

1. **配置驱动**：敌机属性、武器等级等通过配置定义
2. **事件系统**：解耦模块间通信（如分数变化）
3. **插件式**：新道具、新敌机易于添加
